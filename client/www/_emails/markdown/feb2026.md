**Platform API Guide**

Ever wanted to create apps programmatically?

If you're building an app builder, you can provision backends for your users in milliseconds. Or if you're working with a team, you can improve your dev workflows: create apps for tests or spin up an app for every preview PR.

Previously instructions for this were a bit hard to come by, so we wrote a guide. It walks you through how to create apps and how to make changes to them. [Check out the docs](https://www.instantdb.com/docs/platform-api) to learn more!

**SSR for NextJS**

![](https://www.instantdb.com/img/docs/next-ssr-diagram.png)

Building on Cookie Sync, we also shipped SSR support! With the new `useSuspenseQuery` hook, you can render your apps on the server and seamlessly transition to realtime data when the page loads.

Why did we do this?

SSR is great, but nothing beats the speed of local data. Thankfully, you get the best of both worlds: if the client already has the data, we use it. No loading states, no null/undefined data, and still just as fast as before. [Try it out](https://www.instantdb.com/docs/next-ssr) and let us know what you think!
